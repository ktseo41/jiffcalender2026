# 메타 카탈로그

작성일: 2026-04-10  
최종 업데이트: 2026-04-10

## 목적
- 현재 프로젝트 `jiff-schedule-timeline`의 문서와 핵심 파일 위치를 단일 문서에서 관리한다.
- 다른 프로젝트에서 복사해 온 문서 구조는 그대로 남기지 않고, 현재 프로젝트 맥락에 맞게 정리된 항목만 카탈로그에 남긴다.
- 현재 기준 구현 파일은 `jiff2026.html`이며, 이후 문서도 이 파일과 실제 저장소 구조를 기준으로 작성한다.

## 프로젝트 개요
- 프로젝트명: `jiff-schedule-timeline`
- 작업 기준 파일: `jiff2026.html`
- 현재 산출물 성격: JIFF 2026 상영 시간표 단일 HTML 기반 프로토타입

## 네이밍 규칙
- 문서 패턴: `{topic}--{type}.md`
- 스크립트 패턴: `{action}-{target}.js`
- `docs/`: 프로젝트 문서 보관 경로
- `scripts/`: 재사용 가능한 로컬 스크립트 보관 경로
- 예외: `AGENTS.md`는 작업 지침 문서라 루트 경로(`/AGENTS.md`)에 유지

## 문서 운영 규칙
- 새 문서를 추가하면 이 카탈로그에 파일명, 설명, 위치를 함께 등록한다.
- 현재 프로젝트와 무관한 다른 프로젝트 문서나 이력은 남기지 않는다.
- 문서 내용과 실제 파일 구조가 다르면 실제 저장소 상태에 맞춰 이 카탈로그를 먼저 갱신한다.

| --- | --- | --- | --- | --- | --- |

## 현재 문서 상태
- `docs/` 폴더에는 아직 `meta--catalog.md`만 존재한다.
- 다른 프로젝트에서 복사해 온 문서 목록과 변경 이력은 현재 프로젝트와 무관하므로 제거했다.
- 이후 설계, 계획, 메모 문서가 생기면 현재 프로젝트 범위 안에서만 추가한다.

## 변경 이력
- 2026-04-10: 다른 프로젝트용 메타 카탈로그를 현재 프로젝트 `jiff-schedule-timeline` 기준으로 정리하고, 기존 문서 목록과 변경 이력을 제거했다.
- 2026-04-10: 현재 저장소 기준 핵심 파일인 `AGENTS.md`, `jiff2026.html`, `docs/meta--catalog.md`만 카탈로그에 등록했다.
